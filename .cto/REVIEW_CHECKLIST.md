# Review Checklist

## Before Starting a Phase

- [ ] Read `.cto/RULEBOOK.md` + relevant specialized rules  
- [ ] Read `docs/adr/` for related decisions  
- [ ] Phase spec written and **human approved**  

## Before Completing a Phase

- [ ] All phase docs in `docs/phase-{N}/`  
- [ ] `CTO_REPORT.md` with:
  - Architecture Review  
  - Security Review  
  - Database Review  
  - SEO Review  
  - Risks  
  - Technical Debt  
  - Recommendations  
- [ ] `docs/reviews/ARCHITECTURE_GUARDIAN.md` — 5 guardian questions answered  
- [ ] Conventional Commit(s)  
- [ ] Push to `develop` (or feature branch → PR → develop)  
- [ ] **Stop — wait for CTO human review**  

## CTO Decision

- [ ] APPROVE  
- [ ] APPROVE WITH CONDITIONS  
- [ ] REJECT
